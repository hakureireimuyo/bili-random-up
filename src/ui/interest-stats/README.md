# Interest Stats

兴趣分析页面，展示用户兴趣分析结果。

## 功能说明

本页面展示基于用户观看行为分析得到的兴趣数据，包括：

### 1. Top 兴趣标签
- 展示用户兴趣度最高的标签
- 显示综合兴趣分数、短期分数、长期分数
- 展示兴趣趋势（上升/平稳/下降）
- 使用进度条可视化兴趣分数

### 2. 兴趣趋势
- 展示Top 10标签的兴趣变化趋势
- 使用折线图展示历史数据
- 显示当前兴趣分数
- 支持自定义时间范围（默认30天）

### 3. 兴趣星球
- 可视化展示兴趣节点
- 显示节点权重
- 使用图标和颜色区分不同兴趣
- 支持节点交互

## 使用说明

### 初始化
页面加载时会自动初始化兴趣统计组件，首次使用时需要点击"初始化"按钮来初始化兴趣系统。

### 刷新数据
点击"刷新"按钮可以重新加载兴趣数据，数据会从后台获取最新的统计信息。

### 数据来源
所有兴趣数据都来自后台的InterestManager，通过以下方式更新：
- 实时更新：用户观看视频或互动时自动更新
- 定时更新：每日自动更新所有兴趣分数和趋势
- 手动更新：用户可以手动触发定时任务

## 技术实现

### 组件设计
- **InterestStats**: 核心组件类，负责数据获取和渲染
- **InterestStatsConfig**: 配置接口，支持自定义参数
- **模块化**: 组件与页面入口分离，便于维护

### 数据访问
- 通过 `database/implementations` 层访问数据
- 不直接操作IndexedDB，确保数据链路统一
- 使用Repository模式封装数据访问逻辑

### 性能优化
- 懒加载：切换到标签页时才初始化组件
- 批量操作：使用批量查询减少数据库访问
- 缓存机制：避免重复计算和渲染

## 注意事项

1. 首次使用需要初始化兴趣系统
2. 兴趣数据需要一定的观看历史积累
3. 趋势分析需要至少7天的历史数据
4. 兴趣星球数据需要先构建兴趣图
5. 所有操作都是异步的，不会阻塞用户界面
