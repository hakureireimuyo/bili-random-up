# 搜索功能使用说明（前端页面视角）

## 概述

本文档从前端开发者的角度，说明如何使用搜索功能。搜索功能采用"书（Book）"作为数据容器，与页面生命周期绑定，实现高效的复杂搜索和分页功能。

## 核心概念

### 1. Book（书）- 数据容器

Book是页面的数据容器，每个搜索页面对应一个Book：

- **职责**：存储查询结果ID列表和分页数据
- **生命周期**：与页面相同，页面创建时创建，页面销毁时删除
- **特点**：只负责存储数据，不关心查询逻辑

### 2. QueryService - 查询服务

QueryService是全局共享的查询工具：

- **职责**：执行查询逻辑，返回结果ID列表
- **生命周期**：应用启动时创建，全局单例
- **特点**：可被多个页面复用，查询结果在页面间共享

### 3. BookManager - 书管理器

BookManager管理所有页面的Book：

- **职责**：提供Book的创建、获取、删除等操作
- **生命周期**：应用启动时创建，全局单例
- **特点**：管理多个页面的Book，不自动清理

## 快速开始

### 1. 初始化全局服务

在应用启动时初始化全局服务：

```typescript
import { GlobalSearchService } from './search-service';

// 获取全局搜索服务（单例）
const searchService = GlobalSearchService.getInstance();
const bookManager = searchService.getBookManager();
const queryService = searchService.getQueryService();
```

### 2. 创建搜索页面

```typescript
import { SearchPage } from './search-page';

// 创建搜索页面
const searchPage = new SearchPage(
  'page-1',  // 页面ID，唯一标识
  {
    type: 'composite',
    keyword: '游戏',
    tagExpressions: [],
    isFollowing: false,
    platform: 'bilibili'
  },
  bookManager,
  queryService
);
```

### 3. 使用搜索页面

```typescript
// 页面挂载
await searchPage.onMount();

// 加载数据
const creators = await searchPage.loadPage(0);

// 获取分页信息
const pagination = searchPage.getPaginationInfo();

// 更新搜索条件
await searchPage.updateSearchCondition({
  type: 'composite',
  keyword: '音乐',
  tagExpressions: [],
  isFollowing: false,
  platform: 'bilibili'
});

// 页面卸载
searchPage.onUnmount();
```

## React组件示例

### 基础组件

```typescript
import { useEffect, useState } from 'react';
import { GlobalSearchService } from './search-service';
import { SearchPage } from './search-page';

function SearchPageComponent({ pageId, initialCondition }) {
  const [page, setPage] = useState(null);
  const [creators, setCreators] = useState([]);
  const [pagination, setPagination] = useState({
    currentPage: 0,
    totalPages: 0,
    totalRecords: 0,
    pageSize: 20
  });
  const [loading, setLoading] = useState(false);

  useEffect(() => {
    // 页面挂载
    const searchService = GlobalSearchService.getInstance();
    const searchPage = new SearchPage(
      pageId,
      initialCondition,
      searchService.getBookManager(),
      searchService.getQueryService()
    );

    searchPage.onMount()
      .then(() => {
        setPage(searchPage);
        setCreators(searchPage.getCurrentPageData());
        setPagination(searchPage.getPaginationInfo());
      })
      .catch(console.error);

    // 页面卸载
    return () => {
      if (searchPage) {
        searchPage.onUnmount();
      }
    };
  }, [pageId, initialCondition]);

  const handlePageChange = async (newPage: number) => {
    if (!page || loading) return;

    setLoading(true);
    try {
      const data = await page.loadPage(newPage);
      setCreators(data);
      setPagination(page.getPaginationInfo());
    } catch (error) {
      console.error('加载页面失败:', error);
    } finally {
      setLoading(false);
    }
  };

  const handleSearch = async (newCondition) => {
    if (!page || loading) return;

    setLoading(true);
    try {
      await page.updateSearchCondition(newCondition);
      setCreators(page.getCurrentPageData());
      setPagination(page.getPaginationInfo());
    } catch (error) {
      console.error('搜索失败:', error);
    } finally {
      setLoading(false);
    }
  };

  return (
    <div>
      {/* 搜索框 */}
      <input
        type="text"
        placeholder="搜索创作者..."
        onChange={(e) => handleSearch({
          ...initialCondition,
          keyword: e.target.value
        })}
      />

      {/* 加载状态 */}
      {loading && <div>加载中...</div>}

      {/* 创作者列表 */}
      <ul>
        {creators.map(creator => (
          <li key={creator.creatorId}>
            {creator.name}
          </li>
        ))}
      </ul>

      {/* 分页控件 */}
      <div>
        <button
          onClick={() => handlePageChange(pagination.currentPage - 1)}
          disabled={pagination.currentPage === 0 || loading}
        >
          上一页
        </button>
        <span>
          第 {pagination.currentPage + 1} / {pagination.totalPages} 页
        </span>
        <button
          onClick={() => handlePageChange(pagination.currentPage + 1)}
          disabled={pagination.currentPage >= pagination.totalPages - 1 || loading}
        >
          下一页
        </button>
      </div>
    </div>
  );
}
```

## SearchPage API

### 构造函数

```typescript
constructor(
  pageId: string,              // 页面ID，唯一标识
  queryCondition: QueryCondition,  // 查询条件
  bookManager: BookManager,    // 书管理器
  queryService: QueryService   // 查询服务
)
```

### 方法

#### onMount(): Promise<void>
页面挂载时调用，初始化Book并加载第一页数据。

```typescript
await searchPage.onMount();
```

#### loadPage(pageNumber: number): Promise<Creator[]>
加载指定页的数据。

```typescript
const creators = await searchPage.loadPage(0);  // 加载第一页
```

#### updateSearchCondition(newCondition: QueryCondition): Promise<void>
更新搜索条件，重新查询并加载数据。

```typescript
await searchPage.updateSearchCondition({
  type: 'composite',
  keyword: '音乐',
  tagExpressions: [],
  isFollowing: false,
  platform: 'bilibili'
});
```

#### getCurrentPageData(): Creator[]
获取当前页的数据。

```typescript
const creators = searchPage.getCurrentPageData();
```

#### getPaginationInfo(): PaginationInfo
获取分页信息。

```typescript
const pagination = searchPage.getPaginationInfo();
// 返回: { currentPage, totalPages, totalRecords, pageSize }
```

#### onUnmount(): void
页面卸载时调用，删除Book，释放资源。

```typescript
searchPage.onUnmount();
```

## 查询条件类型

### 基础查询条件

```typescript
interface QueryCondition {
  type: 'composite' | 'name' | 'tag';
  keyword?: string;              // 关键词
  tagExpressions?: TagExpression[];  // 标签表达式
  isFollowing?: boolean;         // 是否关注
  platform: string;              // 平台
}
```

### 标签表达式

```typescript
interface TagExpression {
  tagId: string;
  operator: 'AND' | 'OR' | 'NOT';
}
```

### 示例

```typescript
// 关键词搜索
{
  type: 'composite',
  keyword: '游戏',
  platform: 'bilibili'
}

// 标签搜索
{
  type: 'composite',
  tagExpressions: [
    { tagId: 'tag1', operator: 'AND' },
    { tagId: 'tag2', operator: 'OR' }
  ],
  platform: 'bilibili'
}

// 组合搜索
{
  type: 'composite',
  keyword: '游戏',
  tagExpressions: [
    { tagId: 'tag1', operator: 'AND' }
  ],
  isFollowing: false,
  platform: 'bilibili'
}
```

## 最佳实践

### 1. 页面ID管理

使用有意义的页面ID，便于调试和管理：

```typescript
// 好的做法
const pageId = `search-${userId}-${Date.now()}`;

// 不好的做法
const pageId = 'page-1';  // 可能冲突
```

### 2. 错误处理

始终处理异步操作的错误：

```typescript
try {
  await searchPage.onMount();
} catch (error) {
  console.error('页面初始化失败:', error);
  // 显示错误提示
}
```

### 3. 加载状态

管理加载状态，提升用户体验：

```typescript
const [loading, setLoading] = useState(false);

const handlePageChange = async (page) => {
  if (loading) return;  // 防止重复请求

  setLoading(true);
  try {
    await searchPage.loadPage(page);
  } finally {
    setLoading(false);
  }
};
```

### 4. 资源清理

确保在页面卸载时清理资源：

```typescript
useEffect(() => {
  const searchPage = new SearchPage(...);

  return () => {
    searchPage.onUnmount();  // 清理Book
  };
}, []);
```

### 5. 缓存利用

利用全局缓存，避免重复查询：

```typescript
// 查询结果会被缓存
await searchPage.updateSearchCondition(condition1);

// 切换到另一个条件，再切回来，会使用缓存
await searchPage.updateSearchCondition(condition2);
await searchPage.updateSearchCondition(condition1);  // 使用缓存
```

## 常见问题

### Q: 多个搜索页面可以同时存在吗？

A: 可以。每个页面有独立的Book，通过不同的pageId区分。QueryService和BookManager是全局共享的。

### Q: 页面刷新后数据还在吗？

A: 不在。Book存储在内存中，页面刷新后会重新创建。如果需要持久化，需要额外实现。

### Q: 如何处理大量数据？

A: Book使用分页机制，每页数据按需加载。可以通过调整pageSize优化性能：

```typescript
const searchPage = new SearchPage(
  pageId,
  condition,
  bookManager,
  queryService
);
// 默认每页20条，可以调整
```

### Q: 搜索条件变化频繁怎么办？

A: QueryService会缓存查询结果，相同条件不会重复查询。频繁变化条件时，考虑使用防抖：

```typescript
import { debounce } from 'lodash';

const debouncedSearch = debounce(async (condition) => {
  await searchPage.updateSearchCondition(condition);
}, 300);
```

## 性能优化

### 1. 预加载

启用预加载，提升用户体验：

```typescript
const result = await bookManager.getPage(
  bookId,
  pageNumber,
  {
    preloadNext: true,   // 预加载下一页
    preloadCount: 1      // 预加载页数
  }
);
```

### 2. 缓存配置

调整缓存大小和过期时间：

```typescript
const dataCache = new DataCache({
  maxSize: 2000,         // 最大缓存条数
  maxAge: 3600000,       // 过期时间（毫秒）
  cleanupRatio: 0.2      // 清理比例
});
```

### 3. 批量操作

批量获取数据，减少请求次数：

```typescript
// Book内部会批量获取未缓存的数据
const result = await bookManager.getPage(bookId, pageNumber);
```

## 总结

从前端页面的角度来看：

1. **Book是数据容器**：与页面生命周期绑定，存储查询结果和分页数据
2. **QueryService是查询工具**：全局共享，执行查询逻辑
3. **BookManager是管理器**：全局共享，管理所有页面的Book
4. **生命周期清晰**：onMount创建，使用期间操作，onUnmount销毁
5. **性能优化**：利用缓存、预加载、批量操作等机制

通过合理使用这些组件，可以实现高效、易维护的搜索功能。
