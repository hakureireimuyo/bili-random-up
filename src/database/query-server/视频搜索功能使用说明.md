# 视频搜索功能使用说明（前端页面视角）

## 概述

本文档从前端开发者的角度，说明如何使用视频搜索功能。视频搜索功能采用"书（Book）"作为数据容器，与页面生命周期绑定，实现高效的复杂视频搜索和分页功能。

## 核心概念

### 1. VideoBook（视频书）- 数据容器

VideoBook是页面的数据容器，每个视频搜索页面对应一个VideoBook：

- **职责**：存储视频查询结果ID列表和分页数据
- **生命周期**：与页面相同，页面创建时创建，页面销毁时删除
- **特点**：只负责存储数据，不关心查询逻辑

### 2. VideoQueryService - 视频查询服务

VideoQueryService是全局共享的查询工具：

- **职责**：执行视频查询逻辑，返回结果ID列表
- **生命周期**：应用启动时创建，全局单例
- **特点**：可被多个页面复用，查询结果在页面间共享

### 3. VideoBookManager - 视频书管理器

VideoBookManager管理所有页面的VideoBook：

- **职责**：提供VideoBook的创建、获取、删除等操作
- **生命周期**：应用启动时创建，全局单例
- **特点**：管理多个页面的VideoBook，不自动清理

### 4. VideoIndex - 视频索引

VideoIndex是用于快速查询的轻量级数据结构：

- **包含字段**：
  - videoId: 视频ID
  - platform: 平台类型
  - creatorId: 创作者ID
  - title: 视频标题
  - duration: 视频时长（秒）
  - publishTime: 发布时间
  - tags: 视频标签列表
  - isInvalid: 是否失效

## 快速开始

### 1. 初始化全局服务

在应用启动时初始化全局服务：

```typescript
import { GlobalVideoSearchService } from './video-search-service';

// 获取全局视频搜索服务（单例）
const videoSearchService = GlobalVideoSearchService.getInstance();
const videoBookManager = videoSearchService.getVideoBookManager();
const videoQueryService = videoSearchService.getVideoQueryService();
```

### 2. 创建视频搜索页面

```typescript
import { VideoSearchPage } from './video-search-page';

// 创建视频搜索页面
const videoSearchPage = new VideoSearchPage(
  'page-1',  // 页面ID，唯一标识
  {
    platform: 'bilibili',
    keyword: '游戏',  // 标题关键词
    creatorName: '老番茄',  // 创作者名称
    tagExpressions: [
      { tagId: 'game', operator: 'AND' }
    ],
    durationRange: {
      min: 60,   // 最少1分钟
      max: 1800  // 最多30分钟
    },
    onlyFollowingCreators: false
  },
  videoBookManager,
  videoQueryService
);
```

### 3. 使用视频搜索页面

```typescript
// 页面挂载
await videoSearchPage.onMount();

// 加载数据
const videos = await videoSearchPage.loadPage(0);

// 获取分页信息
const pagination = videoSearchPage.getPaginationInfo();

// 更新搜索条件
await videoSearchPage.updateSearchCondition({
  platform: 'bilibili',
  keyword: '音乐',
  tagExpressions: [],
  durationRange: {
    min: 120,
    max: 600
  }
});

// 页面卸载
videoSearchPage.onUnmount();
```

## React组件示例

### 基础组件

```typescript
import { useEffect, useState } from 'react';
import { GlobalVideoSearchService } from './video-search-service';
import { VideoSearchPage } from './video-search-page';

function VideoSearchPageComponent({ pageId, initialCondition }) {
  const [page, setPage] = useState(null);
  const [videos, setVideos] = useState([]);
  const [pagination, setPagination] = useState({
    currentPage: 0,
    totalPages: 0,
    totalRecords: 0,
    pageSize: 20
  });
  const [loading, setLoading] = useState(false);

  useEffect(() => {
    // 页面挂载
    const videoSearchService = GlobalVideoSearchService.getInstance();
    const videoSearchPage = new VideoSearchPage(
      pageId,
      initialCondition,
      videoSearchService.getVideoBookManager(),
      videoSearchService.getVideoQueryService()
    );

    videoSearchPage.onMount()
      .then(() => {
        setPage(videoSearchPage);
        setVideos(videoSearchPage.getCurrentPageData());
        setPagination(videoSearchPage.getPaginationInfo());
      })
      .catch(console.error);

    // 页面卸载
    return () => {
      if (videoSearchPage) {
        videoSearchPage.onUnmount();
      }
    };
  }, [pageId, initialCondition]);

  const handlePageChange = async (newPage: number) => {
    if (!page || loading) return;

    setLoading(true);
    try {
      const data = await page.loadPage(newPage);
      setVideos(data);
      setPagination(page.getPaginationInfo());
    } catch (error) {
      console.error('加载页面失败:', error);
    } finally {
      setLoading(false);
    }
  };

  const handleSearch = async (newCondition) => {
    if (!page || loading) return;

    setLoading(true);
    try {
      await page.updateSearchCondition(newCondition);
      setVideos(page.getCurrentPageData());
      setPagination(page.getPaginationInfo());
    } catch (error) {
      console.error('搜索失败:', error);
    } finally {
      setLoading(false);
    }
  };

  return (
    <div>
      {/* 搜索框 */}
      <input
        type="text"
        placeholder="搜索视频..."
        onChange={(e) => handleSearch({
          ...initialCondition,
          keyword: e.target.value
        })}
      />

      {/* 加载状态 */}
      {loading && <div>加载中...</div>}

      {/* 视频列表 */}
      <ul>
        {videos.map(video => (
          <li key={video.videoId}>
            <h3>{video.title}</h3>
            <p>时长: {Math.floor(video.duration / 60)}分{video.duration % 60}秒</p>
          </li>
        ))}
      </ul>

      {/* 分页控件 */}
      <div>
        <button
          onClick={() => handlePageChange(pagination.currentPage - 1)}
          disabled={pagination.currentPage === 0 || loading}
        >
          上一页
        </button>
        <span>
          第 {pagination.currentPage + 1} / {pagination.totalPages} 页
        </span>
        <button
          onClick={() => handlePageChange(pagination.currentPage + 1)}
          disabled={pagination.currentPage >= pagination.totalPages - 1 || loading}
        >
          下一页
        </button>
      </div>
    </div>
  );
}
```

## VideoQueryCondition 类型

### 基础查询条件

```typescript
interface VideoQueryCondition {
  /** 平台 */
  platform: Platform;
  /** 标题关键词（可选） */
  keyword?: string;
  /** 创作者ID列表（可选） */
  creatorIds?: string[];
  /** 创作者名称（可选，需要通过创作者索引查询） */
  creatorName?: string;
  /** 标签表达式列表（可选） */
  tagExpressions?: TagExpression[];
  /** 时长范围（可选） */
  durationRange?: DurationRange;
  /** 发布时间范围（可选） */
  publishTimeRange?: {
    min?: number;
    max?: number;
  };
  /** 是否只查询已关注的创作者的视频（可选） */
  onlyFollowingCreators?: boolean;
}
```

### 标签表达式

```typescript
interface TagExpression {
  /** 标签ID或标签ID列表（用于OR操作） */
  tagId: string | string[];
  /** 操作符 */
  operator: 'AND' | 'OR' | 'NOT';
}
```

### 时长范围

```typescript
interface DurationRange {
  /** 最小时长（秒） */
  min?: number;
  /** 最大时长（秒） */
  max?: number;
}
```

## VideoSearchPage API

### 构造函数

```typescript
constructor(
  pageId: string,                    // 页面ID，唯一标识
  queryCondition: VideoQueryCondition,  // 查询条件
  videoBookManager: VideoBookManager,  // 视频书管理器
  videoQueryService: VideoQueryService // 视频查询服务
)
```

### 方法

#### onMount(): Promise<void>
页面挂载时调用，初始化VideoBook并加载第一页数据。

```typescript
await videoSearchPage.onMount();
```

#### loadPage(pageNumber: number): Promise<Video[]>
加载指定页的数据。

```typescript
const videos = await videoSearchPage.loadPage(0);  // 加载第一页
```

#### updateSearchCondition(newCondition: VideoQueryCondition): Promise<void>
更新搜索条件，重新查询并加载数据。

```typescript
await videoSearchPage.updateSearchCondition({
  platform: 'bilibili',
  keyword: '音乐',
  tagExpressions: [],
  durationRange: {
    min: 120,
    max: 600
  }
});
```

#### getCurrentPageData(): Video[]
获取当前页的数据。

```typescript
const videos = videoSearchPage.getCurrentPageData();
```

#### getPaginationInfo(): PaginationInfo
获取分页信息。

```typescript
const pagination = videoSearchPage.getPaginationInfo();
// 返回: { currentPage, totalPages, totalRecords, pageSize }
```

#### onUnmount(): void
页面卸载时调用，删除VideoBook，释放资源。

```typescript
videoSearchPage.onUnmount();
```

## 最佳实践

### 1. 页面ID管理

使用有意义的页面ID，便于调试和管理：

```typescript
// 好的做法
const pageId = `video-search-${userId}-${Date.now()}`;

// 不好的做法
const pageId = 'page-1';  // 可能冲突
```

### 2. 错误处理

始终处理异步操作的错误：

```typescript
try {
  await videoSearchPage.onMount();
} catch (error) {
  console.error('页面初始化失败:', error);
  // 显示错误提示
}
```

### 3. 加载状态

管理加载状态，提升用户体验：

```typescript
const [loading, setLoading] = useState(false);

const handlePageChange = async (page) => {
  if (loading) return;  // 防止重复请求

  setLoading(true);
  try {
    await videoSearchPage.loadPage(page);
  } finally {
    setLoading(false);
  }
};
```

### 4. 资源清理

确保在页面卸载时清理资源：

```typescript
useEffect(() => {
  const videoSearchPage = new VideoSearchPage(...);

  return () => {
    videoSearchPage.onUnmount();  // 清理VideoBook
  };
}, []);
```

### 5. 缓存利用

利用全局缓存，避免重复查询：

```typescript
// 查询结果会被缓存
await videoSearchPage.updateSearchCondition(condition1);

// 切换到另一个条件，再切回来，会使用缓存
await videoSearchPage.updateSearchCondition(condition2);
await videoSearchPage.updateSearchCondition(condition1);  // 使用缓存
```

## 常见问题

### Q: 多个视频搜索页面可以同时存在吗？

A: 可以。每个页面有独立的VideoBook，通过不同的pageId区分。VideoQueryService和VideoBookManager是全局共享的。

### Q: 页面刷新后数据还在吗？

A: 不在。VideoBook存储在内存中，页面刷新后会重新创建。如果需要持久化，需要额外实现。

### Q: 如何处理大量视频数据？

A: VideoBook使用分页机制，每页数据按需加载。可以通过调整pageSize优化性能：

```typescript
const videoSearchPage = new VideoSearchPage(
  pageId,
  condition,
  videoBookManager,
  videoQueryService
);
// 默认每页20条，可以调整
```

### Q: 搜索条件变化频繁怎么办？

A: VideoQueryService会缓存查询结果，相同条件不会重复查询。频繁变化条件时，考虑使用防抖：

```typescript
import { debounce } from 'lodash';

const debouncedSearch = debounce(async (condition) => {
  await videoSearchPage.updateSearchCondition(condition);
}, 300);
```

### Q: 如何基于创作者名称搜索视频？

A: VideoQueryService会自动通过创作者索引缓存查询创作者ID，然后返回该创作者的视频：

```typescript
{
  platform: 'bilibili',
  creatorName: '老番茄'
}
```

调整缓存大小和过期时间：

```typescript
const dataCache = new DataCache({
```

## 总结

从前端页面的角度来看：

1. **VideoBook是数据容器**：与页面生命周期绑定，存储查询结果和分页数据
2. **VideoQueryService是查询工具**：全局共享，执行查询逻辑
3. **VideoBookManager是管理器**：全局共享，管理所有页面的VideoBook
4. **生命周期清晰**：onMount创建，使用期间操作，onUnmount销毁
5. **性能优化**：利用缓存、预加载、批量操作等机制

通过合理使用这些组件，可以实现高效、易维护的视频搜索功能。
