
# Repository 层职责说明

## 一、定位

Repository 层是系统中的**数据中枢与唯一数据入口**，负责统一管理：

* 数据库访问
* 缓存读写
* 查询执行协调

其本质角色是：

> **在“数据存储（Database）”与“业务查询（Query / Book）”之间建立一个稳定、可控的中间层**

---

## 二、核心职责

## 1. 数据访问职责（Data Access）

* 封装所有数据库操作（增删改查）
* 对外提供统一的数据接口
* 隔离底层数据库实现细节
* 确保上层不直接依赖数据库

---

## 2. 缓存管理职责（Cache Control）

* **唯一允许修改缓存的模块**
* 统一管理：

  * IndexCache
  * TagCache
  * DataCache
* 控制缓存的更新时机与策略
* 避免缓存被多处写入导致状态不一致

---

## 3. 数据一致性职责（Consistency Control）

* 在数据库与缓存之间建立一致性保障机制
* 约束写操作流程（先数据库，后缓存）
* 在异常情况下进行降级处理（如缓存失效）
* 避免出现“查询结果与真实数据不一致”的情况

---

## 4. 查询协调职责（Query Coordination）

* 接收上层的查询请求
* 调度缓存（尤其是 TagCache 与 IndexCache）完成查询
* 执行查询流程控制（而不是具体算法实现）
* 返回标准化的结果（ID 列表或数据对象）

---

## 5. 数据转换职责（Data Transformation）

* 在不同数据表示之间进行转换：

  * index ↔ id
  * id ↔ 完整对象
* 屏蔽底层数据结构细节
* 保证上层使用统一的数据格式

---

## 6. 查询入口职责（Query Gateway）

* 提供统一的查询接口
* 支持：

  * 基于标签的组合查询
  * 基于索引字段的过滤
* 不允许上层直接访问缓存进行查询

---

## 7. 分页与数据获取协调

* 基于 ID 列表提供数据获取能力
* 协调 DataCache 与数据库完成数据加载
* 为 Book 层提供稳定的数据访问方式

---

## 三、职责边界（非常重要）

## Repository **负责的**

* 数据读取与写入
* 调用src\database\implementations实现的基础操作和src\database\query-server中实现的复杂查询
* 缓存更新与维护
* 查询执行的调度
* 数据格式转换
* 一致性保障

---

## Repository **不负责的**

### ❌ 不负责查询逻辑定义

* 不定义“如何过滤”
* 不实现复杂查询算法（由 Query / Engine 完成）

---

### ❌ 不负责缓存结构设计

* 不关心 TagCache 内部如何实现（bitset / array）
* 只负责使用与更新

---

### ❌ 不负责 UI 或分页状态

* 不管理 Book 生命周期
* 不保存查询状态

---

### ❌ 不负责业务逻辑

* 不包含业务规则判断
* 仅处理数据层逻辑

---

## 四、与其他层的关系

---

## 1. 与 Query 层

```text
Query → Repository
```

* Query 定义查询条件
* Repository 执行查询流程
* Query 不直接访问缓存或数据库

---

## 2. 与 Cache 层

```text
Repository → Cache
```

* Repository 是缓存唯一写入口
* Cache 不主动更新自身

---

## 3. 与 Database

```text
Repository → Database
```

* 所有数据库操作必须通过 Repository
* 数据库不直接暴露给上层

---

## 4. 与 Book 层

```text
Book → Repository
```

* Book 通过 Repository 获取数据
* Book 不接触缓存与数据库

---

## 五、设计原则

---

### 1. 单一入口原则

> 所有数据操作必须经过 Repository

---

### 2. 写路径统一原则

> 数据写入/更新/删除必须同时协调数据库与缓存

---

### 3. 读路径稳定原则

> 查询必须基于缓存，避免直接查询数据库

---

### 4. 缓存控制集中原则

> Cache 的更新必须集中在 Repository

---

### 5. 数据结构隔离原则

> 上层不感知 index / cache 等内部结构

---

## 六、系统中的作用总结

Repository 在系统中承担的是：

```text
数据访问层 + 查询执行协调层 + 一致性控制中心
```
