以下为 **Agent 开发规范（Agent Coding Protocol）**。
该规范用于指导智能体自动实现 **Bilibili Discovery Engine 浏览器扩展**，确保：

* 代码结构一致
* 任务执行稳定
* 自动生成代码可持续迭代
* 多 Agent 协作可控

该文档是 **自动化开发系统的核心规范**。

---

# 一、协议目标

Agent Coding Protocol 解决三个问题：

1. **LLM 生成代码不稳定**
2. **代码结构混乱**
3. **模块难以集成**

因此协议规定：

* 统一项目结构
* 统一任务接口
* 统一代码风格
* 统一测试方式

---

# 二、开发流程

Agent 开发流程如下：

```text
任务输入
   │
   ▼
Code Agent
生成代码
   │
   ▼
Test Agent
执行测试
   │
   ▼
Review Agent
检查规范
   │
   ▼
Merge Agent
合并代码
```

每个任务遵循：

```text
Task → Code → Test → Review → Merge
```

---

# 三、Agent角色

系统包含四类 Agent。

| Agent        | 职责     |
| ------------ | ------ |
| Code Agent   | 编写代码   |
| Test Agent   | 生成测试   |
| Review Agent | 代码规范检查 |
| Merge Agent  | 集成代码   |

---

# 四、代码结构规范

所有代码必须遵循以下目录结构。

```text
extension
│
├── manifest.tson
│
├── background
│   └── service-worker.ts
│
├── content
│   └── tracker.ts
│
├── api
│   └── bili-api.ts
│
├── engine
│   ├── classifier.ts
│   └── recommender.ts
│
├── storage
│   └── storage.ts
│
├── ui
│   ├── popup
│   └── options
│
└── utils
```

规则：

```text
每个模块一个文件
```

禁止：

```text
多个模块混合
```

# 六、模块设计规范

每个模块必须包含：

```Typescript
/**
 * Module Description
 */
export function functionName() {

}
```

示例：

```Typescript
/**
 * Fetch videos of a specific UP.
 */
export async function getUPVideos(mid) {

}
```

---

# 七、函数设计规范

每个函数必须满足：

1. 单一职责
2. 明确输入
3. 明确输出

示例：

```Typescript
/**
 * Get videos from a UP.
 *
 * @param {number} mid
 * @returns {Promise<Video[]>}
 */
export async function getUPVideos(mid) {

}
```

---

# 八、数据结构规范

所有核心数据结构必须定义。

示例：

```Typescript
/**
 * @typedef {Object} Video
 * @property {string} bvid
 * @property {string} title
 * @property {number} play
 */
```

---

# 九、错误处理规范

所有 API 调用必须处理错误。

示例：

```Typescript
try {
    const data = await fetch(url)
} catch(err) {
    console.error("API error", err)
}
```

禁止：

```Typescript
await fetch(url)
```

无异常处理。
