下面给出一份**完整的程序设计文档（Software Design Specification）**，用于指导智能体实现一个浏览器扩展：

目标系统：

> **Bilibili Discovery Engine**
> 一个用于探索 Bilibili 内容的浏览器扩展，支持
>
> * 随机访问关注 UP
> * 自动分类 UP
> * 跳转到视频
> * 基于观看行为建立用户兴趣模型
> * 推荐可能感兴趣的视频

目标平台：

* Google Chrome
* Microsoft Edge
* Manifest V3

---

# 一、系统目标

## 1.1 核心目标

系统主要解决三个问题：

1. **探索内容**
2. **组织关注 UP**
3. **推荐用户可能喜欢的视频**

系统能力：

| 能力    | 说明        |
| ----- | --------- |
| 随机 UP | 随机访问关注UP  |
| 随机视频  | 从UP随机选择视频 |
| UP分类  | 自动分类UP    |
| 兴趣建模  | 分析用户观看行为  |
| 推荐系统  | 推荐潜在兴趣视频  |

---

# 五、推荐系统

推荐系统分三层：

```

↓

↓

```

---

## 5.1 用户兴趣模型

记录用户观看行为：

| 数据   | 用途   |
| ---- | ---- |
| 视频标签 | 兴趣领域 |
| 观看时长 | 兴趣强度 |
| 观看次数 | 偏好   |

---

数据结构：

```json
UserInterest {
  tag: string
  score: number
}
```

示例：

```json
[
 {"tag":"AI","score":10},
 {"tag":"编程","score":8},
 {"tag":"历史","score":2}
]
```

---

## 5.2 兴趣更新算法

公式：

```
score += watch_time / video_duration
```

示例：

```
观看 80% 视频
score += 0.8
```

---

## 5.3 推荐UP

计算：

```
UP得分 = Σ(UP标签 ∩ 用户兴趣)
```

示例：

```
用户兴趣
AI 10
编程 8

UP标签
编程
AI

UP得分 = 18
```

---

## 5.4 推荐视频

评分函数：

```
score =
   interest_weight
 + popularity_weight
 + freshness_weight
```

权重：

```
兴趣 0.6
播放量 0.2
发布时间 0.2
```
