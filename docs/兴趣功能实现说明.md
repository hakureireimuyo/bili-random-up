# 兴趣功能实现说明

## 实现概述

本文档说明兴趣分析系统的实现情况，包括已完成的功能和后续建议。

## 已完成的功能

### 1. 核心计算模块

#### 1.1 InterestCalculator（兴趣计算器）
- **位置**: `src/database/implementations/interest-calculator.impl.ts`
- **功能**:
  - 计算单个标签的兴趣分数（短期、长期、综合）
  - 批量计算多个标签的兴趣分数
  - 重新计算所有标签的兴趣分数
  - 应用时间衰减算法
  - 计算兴趣趋势（上升/平稳/下降）
- **配置参数**:
  - 短期兴趣时间窗口（默认7天）
  - 时间衰减系数（默认0.1）
  - 行为权重配置（观看时长、完整观看、互动）
  - 短期和长期兴趣权重

#### 1.2 InterestGraphBuilder（兴趣图构建器）
- **位置**: `src/database/implementations/interest-graph-builder.impl.ts`
- **功能**:
  - 基于分类构建兴趣图
  - 基于标签相似度构建兴趣图
  - 创建和更新兴趣节点
  - 计算节点权重
  - 关联相似节点
  - 重建兴趣图
- **配置参数**:
  - 标签相似度阈值（默认0.7）
  - 节点权重计算方式（sum/avg/max）
  - 是否自动创建节点
  - 节点颜色配置
  - 节点图标配置

#### 1.3 InterestTrendAnalyzer（趋势分析器）
- **位置**: `src/database/implementations/interest-trend-analyzer.impl.ts`
- **功能**:
  - 分析单个标签的兴趣趋势
  - 批量分析多个标签的趋势
  - 分析所有标签的趋势
  - 计算移动平均
  - 计算变化率
  - 判断趋势类型（上升/平稳/下降）
  - 计算置信度
  - 预测未来趋势
  - 获取趋势摘要
  - 对比多个标签的趋势
- **配置参数**:
  - 分析时间窗口（默认30天）
  - 移动平均窗口大小（默认7）
  - 趋势判断阈值
  - 预测天数（默认7天）

### 3. 后台管理模块

#### 3.1 InterestManager（兴趣管理器）
- **位置**: `src/background/modules/interest-manager.ts`
- **功能**:
  - 处理观看事件，触发兴趣计算
  - 处理互动事件，触发兴趣计算
  - 每日定时任务：更新所有兴趣分数、更新兴趣图权重、分析趋势
  - 每周定时任务：重建兴趣图
  - 每月定时任务：清理过期历史数据、删除低权重节点
  - 获取兴趣统计信息
  - 初始化兴趣系统
- **配置参数**:
  - 是否启用自动计算
  - 是否启用图构建
  - 是否启用趋势分析
  - 历史数据保留天数（默认365天）

#### 3.2 定时任务
- **位置**: `src/background/modules/alarms.ts`
- **已注册的定时任务**:
  - `ALARM_DAILY_INTEREST`: 每日兴趣更新（24小时）
  - `ALARM_WEEKLY_INTEREST`: 每周兴趣图重建（7天）
  - `ALARM_MONTHLY_INTEREST`: 每月数据清理（30天）

#### 3.3 消息处理
- **位置**: `src/background/modules/messages.ts`
- **支持的消息类型**:
  - `get_interest_stats`: 获取兴趣统计信息
  - `initialize_interest_system`: 初始化兴趣系统
  - `run_daily_interest_task`: 手动触发每日任务
  - `run_weekly_interest_task`: 手动触发每周任务
  - `run_monthly_interest_task`: 手动触发每月任务
  - `watch_progress`: 观看进度更新时自动触发兴趣计算

## 数据流转

### 实时更新流程
```
用户观看视频 → 记录WatchEvent
          → 触发InterestManager.handleWatchEvent
          → InterestCalculator计算兴趣分数
          → 更新InterestScore
          → 记录InterestHistory
          → 更新InterestNode权重
```

### 定时任务流程
```
每日任务 → InterestCalculator.recalculateAllInterestScores
         → InterestGraphBuilder.updateNodeWeights
         → InterestTrendAnalyzer.analyzeAllTrends

每周任务 → InterestGraphBuilder.rebuildGraph

每月任务 → 清理过期InterestHistory
         → 删除低权重InterestNode
```

## 技术特点

1. **模块化设计**: 各模块职责清晰，易于维护和扩展
2. **配置驱动**: 支持灵活配置各种参数
3. **性能优化**: 批量操作、增量更新、懒加载
4. **数据一致性**: 使用事务确保批量操作的原子性
5. **自动维护**: 定时任务自动更新和清理数据
6. **用户友好**: 提供可视化展示和手动触发功能

## 注意事项

1. 所有数据操作都通过`implementations`层完成，遵循代码规范
2. 兴趣计算是异步的，不会阻塞用户操作
3. 定时任务在后台自动执行，无需用户干预
4. 历史数据会定期清理，避免占用过多存储空间
5. 用户可以随时手动触发定时任务，方便测试和调试
